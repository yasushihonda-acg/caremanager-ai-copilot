# ADR 0009: ステージベース開発モデルへの移行

## ステータス
Accepted

## コンテキスト

MVPが完了しデモ版が動作中だが、現在のロードマップは**機能カテゴリ別**（品質向上/記録効率化/連携強化/業務最適化）の4フェーズで構成されている。この構成には以下の問題がある：

### 旧モデル（Phase 1-4）の問題点

1. **進捗の不正確な表現**: Phase 2（記録業務効率化）やPhase 3（連携機能強化）が「✅完了」と表記されているが、各フェーズ内に未実装タスクが残っている（音声→構造化変換、議事録自動生成、PDF出力など）
2. **開発ステージとの乖離**: 機能カテゴリ別のフェーズは「何を作るか」を示すが、「いつ本番投入できるか」「次に何をすべきか」の判断基準にならない
3. **優先順位の曖昧さ**: P0/P1/P2の区別はあるが、パイロット投入に必要な最低限のスコープが明確でない

### 新モデル（Stage 1-4）の定義

| Stage | 名称 | 目的 | 退出基準 |
|-------|------|------|---------|
| 1 | MVP Foundation | 全機能のEnd-to-End動作確認 | デモ可能な状態 |
| 2 | Production Readiness | 品質保証・精度検証 | P0完了、AI精度85%+実証 |
| 3 | Pilot Deployment | 実ユーザーでの検証 | 3-5名、満足度80%+ |
| 4 | Scale & Enhancement | フィードバック反映・新機能 | Go/No-Go判定 |

## 決定

**ステージベース開発モデル**を採用する。

### 旧フェーズとの対応

| 旧フェーズ | 旧タスク | 新ステージ |
|-----------|---------|-----------|
| Phase 1 完了分 | 文例DB、V2 API、テスト基盤 | Stage 1（完了） |
| Phase 1 残り | 整合性チェック、抽出ルール最適化 | Stage 2 P0/P1 |
| Phase 2 完了分 | モニタリング、支援経過 | Stage 1（完了） |
| Phase 2 残り | 音声→構造化変換 | Stage 2 P2（延期） |
| Phase 3 完了分 | 第4表、入院時連携 | Stage 1（完了） |
| Phase 3 残り | 議事録自動生成、PDF出力 | Stage 2 P2（延期） |
| Phase 4 | 給付管理サポート | Stage 4（将来） |

### 延期判断の基準

「パイロットで手動代替が可能」かつ「実装コストが高い」タスクはStage 2 P2として延期し、パイロットフィードバック後に再判断する。
