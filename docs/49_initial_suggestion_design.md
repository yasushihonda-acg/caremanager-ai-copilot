
# 49_initial_suggestion_design.md - Initial Suggestion Feature Design

## 1. 背景と目的
*   **課題:** アセスメント開始直後はデータが空であるため、AIに解析させても有益なアドバイス（Co-pilot）が出力されるまで時間がかかる、または当たり前の回答しか返ってこない。
*   **目的:** アプリを開いた瞬間から、ベテランケアマネジャーのノウハウに基づいた「導入質問（アイスブレイク〜基本確認）」を表示し、スムーズな面談開始を支援する。

## 2. 基本サジェスト内容 (Hardcoded Advice)
ベテランケアマネジャー（佐藤氏）監修のもと、以下の3点を初期表示する。

1.  **体調確認:** 「本日の体調はいかがですか？（顔色、バイタル、気分の確認）」
2.  **サービス利用状況:** 「現在利用している介護サービスや医療サービスはありますか？（v1.1.0追加項目の確認）」
3.  **主訴の確認:** 「今、生活の中で一番困っていること、不安なことは何ですか？」

## 3. 動作仕様
*   **初期表示:** `TouchAssessment` コンポーネントがマウントされた時点で上記リストを表示する。
*   **録音開始後:**
    *   **新規 (New):** 初期リストを表示したまま録音開始。最初のAI解析結果（約30秒後）が返ってきた時点で、AI生成のアドバイスに上書き更新する。
    *   **再開 (Resume):** 既存のアドバイスがあればそれを維持する。なければ初期リストを表示する。

## 4. UI/UX効果
*   ユーザーは「何を話そうか」と迷うことなく、画面に表示されたスクリプトを読み上げるだけでアセスメントを開始できる。
*   APIレイテンシ（待ち時間）による「空白の時間」を埋めることができる。
