# 実装計画: ケアプラン第1表・第2表 A4横向き1ページフィット修正

## 問題の概要

スクリーンショットで確認された問題:
1. **第1表**: テーブルがページ上部に圧縮、下半分が空白。「意向」「援助方針」等の大きな記入欄の高さが不足
2. **第2表**: 同様にテーブルが上部に圧縮、下半分が空白。データ行が少ない場合に空白行で埋めていない
3. **ウォーターマーク**: 「ケアマネのミカタ 出力」テキストが不要

## 公式様式の特徴（厚生労働省標準様式）

### 第1表の行配置（A4横向き 297mm×210mm）
| 行 | 内容 | 高さの比率 |
|----|------|-----------|
| ヘッダー | 保険者番号・タイトル・作成日・初回/継続 | 固定 ~25mm |
| 行1 | 利用者名 / 生年月日 / 住所 | 固定 ~8mm |
| 行2 | 居宅サービス計画作成者氏名 | 固定 ~8mm |
| 行3 | 居宅介護支援事業者・事業所名及び所在地 | 固定 ~8mm |
| 行4 | 計画作成（変更）日 / 初回計画作成日 | 固定 ~8mm |
| 行5 | 認定日 / 認定の有効期間 | 固定 ~8mm |
| 行6 | 要介護状態区分 | 固定 ~8mm |
| **行7** | **利用者及び家族の意向を踏まえた課題分析の結果** | **大 ~45mm** |
| **行8** | **介護認定審査会の意見及びサービスの種類の指定** | **中 ~20mm** |
| **行9** | **総合的な援助の方針** | **大 ~40mm** |
| 行10 | 生活援助中心型の算定理由 | 固定 ~10mm |

### 第2表の構成
- 2段ヘッダー（目標 colSpan=4 / 援助内容 colSpan=6）
- データ行はページ残り全体を使用
- **公式様式は空行が多い**（手書き用）→ データが少なくてもページを埋める

## 修正対象ファイル

- `components/careplan/PrintPreview.tsx` のみ

## 修正内容（3ステップ）

### Step 1: 第1表のA4フィット対応

**アプローチ**: コンテナに固定高さ + テーブル height:100% + キー行に明示的height

1. 外側コンテナ: `minHeight: '194mm'` → `height: '194mm'` に変更、`display: 'flex', flexDirection: 'column'`
2. ヘッダーエリア（保険者番号〜初回/継続）は固定高さ
3. テーブルに `style={{ height: '100%' }}` を追加し `flex: 1` で残り領域を埋める
4. 「意向」行に `height: '45mm'`
5. 「審査会意見」行に `height: '20mm'`
6. 「援助方針」行に `height: '40mm'`
7. 「生活援助理由」行に `height: '10mm'`
8. PRINT_STYLESにも同様のheight指定を追加

### Step 2: 第2表のA4フィット対応

1. 外側コンテナ: 同様に `height: '194mm'` + flex
2. テーブルに `height: '100%'` 追加
3. データ行が少ない場合、**空行（最低8行分）**を追加して公式様式の見た目を再現
4. PRINT_STYLESにも対応

### Step 3: ウォーターマーク削除

- 第1表・第2表・第3表の「ケアマネのミカタ 出力」div を3箇所すべて削除

## 印刷/プレビュー両方の整合性

現在の仕組み:
- **プレビュー**: Reactインラインスタイル → そのまま表示
- **印刷**: `handlePrint()` が `printRef.current.innerHTML` + `PRINT_STYLES` で新規ウィンドウに描画

修正方針:
- インラインスタイルでheight指定すれば、innerHTML経由の印刷にも反映される
- PRINT_STYLESのクラスベースCSSにもfallback指定を追加（二重管理だが安全）

## 検証方法

1. Playwrightでプレビュー画面のスクリーンショットを撮影し、テーブルがページを埋めていることを確認
2. 印刷ダイアログ経由のPDF出力でA4横向き1ページに収まることを確認（目視）
