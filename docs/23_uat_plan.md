# 23_uat_plan.md - 運用テスト (UAT) 計画書

## 1. テスト概要
*   **対象バージョン:** 1.0.0 (Release Candidate)
*   **実施期間:** 2025年11月22日 〜 2025年11月23日
*   **目的:** 開発されたシステムが、実際のケアマネジメント業務フローにおいて実用的であり、かつ2025年の運営指導基準に適合しているかを検証する。

## 2. テスター構成 (ペルソナ)
*   **佐藤さん (ベテランケアマネ - 50代):**
    *   **視点:** 「実地指導で指摘されないか」「法的整合性は完璧か」「文字は読みやすいか」
    *   **重視機能:** コンプライアンスチェック、全体サマリー、文字サイズ。
*   **鈴木さん (新人ケアマネ - 20代):**
    *   **視点:** 「聞き漏らしがないか」「専門用語の使い方は合っているか」
    *   **重視機能:** AIオートマッピング、インタビュー・コパイロット。

## 3. テスト環境
*   **デバイス:** iPad mini (想定), PCブラウザ (Chrome/Tablet Mode)
*   **ネットワーク:** Wi-Fi環境 (オンライン)
*   **データ:** デモ用ダミーデータ (山田太郎様)

## 4. テストシナリオ (Scenarios)

### Scenario A: 新規アセスメントの実施 (Touch & Talk)
1.  **初期状態:** アセスメントデータが空の状態から開始。
2.  **音声入力:** 「リアルタイム解析(30秒)」をオンにして、利用者役と会話する。
    *   *会話例:* 「最近、お風呂で背中を洗うのが大変でね（ADL:一部介助）」「夜になると昔のことを思い出して歩き回っちゃうの（認知症:BPSD）」
3.  **検証:**
    *   会話中にAIがチェックボックスを自動で埋めるか？
    *   「聞き漏らしアドバイス」が適切に表示されるか？
    *   要約テキストが重複せず、きれいに更新されるか？

### Scenario B: ケアプラン作成と法的整合性チェック (The Defense)
1.  **日付入力:** 意図的に「原案作成日」を「アセスメント日」より**前**に設定する。
2.  **検証:**
    *   画面に「法的整合性エラー」が表示されるか？
    *   担当者会議日や同意日についても同様にチェックが働くか？
3.  **目標設定:** AIリファイン機能を使って、曖昧な長期目標を具体化する。

### Scenario C: モニタリングと記録 (Timeline)
1.  **記録作成:** 月次モニタリングを入力する。
2.  **目標連動:** アセスメント結果や設定した短期目標と、モニタリングの評価が連動しているか確認する。
3.  **検証:** 過去の履歴が見やすく表示されるか？

## 5. 合否判定基準 (Pass/Fail Criteria)
*   **Critical:** アプリがクラッシュしないこと。法的日付矛盾がある状態で「エラーなし」と判定されないこと。
*   **Major:** AI自動入力が、会話内容と全く逆の判定（自立なのに全介助など）をしないこと。
*   **Minor:** 誤字脱字、レイアウトの軽微なズレは許容するが記録する。
