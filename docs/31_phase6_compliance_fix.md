# 31_phase6_compliance_fix.md - Phase 6 Compliance Hotfix Plan

## 1. 概要
ベテランケアマネジャーからの指摘に基づき、介護保険制度の「課題分析標準項目（23項目）」に対する不適合（項目不足）を解消する緊急アップデート。

## 2. 追加するデータ項目 (Gap Filling)

### A. 健康・医学的側面 (Health Tab)
1.  **皮膚・褥瘡の状態 (`skinCondition`)**
    *   選択肢: 問題なし, 乾燥・痒みあり, 褥瘡・発赤あり, 医療処置中
2.  **口腔衛生・嚥下 (`oralHygiene`)**
    *   選択肢: 問題なし, 歯科受診が必要, 義歯不適合, 咀嚼・嚥下困難
3.  **水分摂取 (`fluidIntake`)**
    *   選択肢: 十分摂取, 意識して摂取中, 不足気味, 制限あり(医師指示)

### B. 生活機能 (ADL Tab)
4.  **衣服の着脱 (`adlDressing`)**
    *   選択肢: 自立, ボタン等一部介助, 全介助, 季節・TPO不適合

### C. 基本情報/社会 (Health or Social Tab)
5.  **サービス利用状況 (`serviceHistory`)**
    *   選択肢: 初回(利用なし), 介護保険のみ利用中, 医療・障害併用, 自費サービス利用

## 3. 変更対象ファイル
*   `types.ts`: `AssessmentData` インターフェースの拡張。
*   `App.tsx`: 初期データの更新。
*   `services/geminiService.ts`: JSON Schemaへの項目追加（Enum定義含む）。
*   `components/TouchAssessment.tsx`:
    *   新規項目のUIコンポーネント配置。
    *   サマリー表の日本語ラベル更新。

## 4. 検証計画
*   **音声入力テスト:** 「背中が痒くて薬を塗っている（皮膚）」「入れ歯が合わなくて（口腔）」「着替えは手伝ってもらっている（着脱）」といった発話に対し、正しく自動チェックが入るか確認する。
